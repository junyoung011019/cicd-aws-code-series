# 🚀 Spring Boot + AWS & Jenkins CI/CD 파이프라인 마스터 로드맵

본 문서는 스프링 부트(Spring Boot) 웹 애플리케이션을 AWS EC2에 자동 배포하는 파이프라인을 단계별로 구축하며 인프라 구조를 학습하기 위한 가이드라인입니다.

---

## 🗺️ CI/CD 정복을 위한 2단계 로드맵

> ⚠️ AWS 무료 계정에서 CodeCommit 사용 불가 (2024년 이후 신규 계정 제한). GitHub으로 바로 시작합니다.

### 📍 1단계: GitHub + AWS 세트

- **흐름:** `내 컴퓨터` ➔ `GitHub` ➔ `AWS CodeBuild` ➔ `AWS CodeDeploy` ➔ `AWS EC2` _(전체 지휘: CodePipeline)_
- **학습 포인트:** GitHub을 소스 저장소로 연결하고 AWS 네이티브 서비스(CodeBuild, CodeDeploy, CodePipeline)로 자동 배포 파이프라인을 구축합니다. IAM Role과 서비스 간 연동 방식을 이해하는 기초 단계입니다.

### 📍 2단계: 빌드 및 지휘관 교체 (Jenkins 연동)

- **흐름:** `내 컴퓨터` ➔ `GitHub` ➔ `Jenkins (교체)` ➔ `AWS CodeDeploy` ➔ `AWS EC2`
- **학습 포인트:** AWS CodeBuild와 CodePipeline을 모두 걷어내고 오픈소스 최강자인 **Jenkins**를 주인공으로 투입합니다. 젠킨스가 직접 소스코드를 빌드하고, 결과물을 S3에 업로드한 뒤 AWS CodeDeploy에 배포 명령을 내리는 하이브리드 구조를 완성합니다.

---

## 📂 우리가 관리할 핵심 파일 4개

복잡한 비즈니스 로직 코딩은 필요하지 않습니다. VS Code에서 아래 4개의 핵심 파일만 작성하거나 수정하면 전체 파이프라인이 완성됩니다.

| #   | 파일명                    | 위치                   | 역할                                                           | 비고              |
| --- | ---------------------- | -------------------- | ------------------------------------------------------------ | --------------- |
| 1   | `HelloController.java` | `src/main/java/.../` | 웹 화면에 글자(`Version 1`)를 띄워주는 자바 컨트롤러                          | 텍스트 수정 및 배포 확인용 |
| 2   | `buildspec.yml`        | 프로젝트 최상위 (Root)      | **AWS CodeBuild**가 코드를 어떻게 빌드(JAR 생성)할지 적어둔 명세서              | 1, 2단계에서만 사용    |
| 3   | `appspec.yml`          | 프로젝트 최상위 (Root)      | **AWS CodeDeploy**가 파일을 EC2의 어느 경로에 풀고 어떤 스크립트를 실행할지 적어둔 명세서 | **모든 단계 재사용**   |
| 4   | `deploy.sh`            | `scripts/`           | **EC2 서버 내부**에서 기존 스프링 프로세스를 종료하고 새 JAR 파일을 구동하는 쉘 스크립트      | **모든 단계 재사용**   |

> 💡 **Tip:** 3번(`appspec.yml`)과 4번(`deploy.sh`)은 1단계에서 한 번 잘 만들어두면 2, 3단계에서 아무런 수정 없이 **그대로 재사용**할 수 있습니다. 즉, 뒤로 갈수록 세팅 난이도가 낮아집니다.
